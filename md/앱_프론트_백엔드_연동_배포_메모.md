# 앱(프론트·백엔드) 연동과 배포 시 참고 메모

인프라(SSM·ALB·GHCR)와 별개로, **앱 코드를 붙이면서 자주 틀어지는 지점**과 **배포·운영 시 확인할 환경 변수**를 한곳에 모았습니다.  
상세한 트러블슈팅 스토리는 모노레포 루트의 **[`../md/프론트엔드_백엔드_연동_이슈와_해결.md`](../../md/프론트엔드_백엔드_연동_이슈와_해결.md)** (경로: `mini_project_3/md/…`)를 참고하면 됩니다.

## 1. 배포 전에 맞출 환경 변수(대표)

| 구분 | 변수(예) | 메모 |
| --- | --- | --- |
| 프론트 | `NEXT_PUBLIC_API_URL` | Next 빌드 시 번들에 박힘. **이미지 빌드 ARG**로 넣거나, 운영 ALB/내부 DNS에 맞게 CI·Docker에서 설정. |
| 백엔드 | `SPRING_DATASOURCE_URL` / DB·Redis | Aurora·ElastiCache 엔드포인트와 일치하는지. `medical-backend.env`에 반영 후 SSM 재배포. |
| 백엔드 | `JWT_SECRET`, `ENCRYPTION_KEY` | 누락 시 로그인·주민번호 필드 등에서 런타임 오류. |
| 백엔드 | `MAIL_USERNAME` / `MAIL_PASSWORD` | 비밀번호 재설정 메일. `APP_MAIL_DELIVERY=auto`(기본)이면 사용자명이 비어 있을 때 **로그에만 링크** 출력. |
| 백엔드 | `APP_FRONTEND_URL` (`application.yml`의 `app.frontend-base-url`) | 재설정 메일의 **프론트 링크 베이스**; 실제 사용자가 여는 도메인·포트와 맞출 것. |
| 백엔드 | `AI_BASE_URL` | AI ALB DNS와 불일치 시 프록시 타임아웃·비정상 응답. |

## 2. API·데이터 “계약”이 깨질 때(요약)

- **환자 목록**이 비면: 백엔드가 `patient_details`만 보던 시절과 달리, 지금은 **PATIENT 역할 사용자 전체**를 내려주는지·프론트가 동일 DTO를 파싱하는지 확인.
- **`resident_number` / AttributeConverter 오류**: DB에 평문·형식 불일치 시 JPA 로딩 실패 → 백엔드 `EncryptionConverter` 방어 로직·`ENCRYPTION_KEY` 확인.
- **의사·간호사 인증코드**: 프론트 하드코딩이 아니라 **`POST /api/auth/validate-staff-code`** + 최종 **`/api/auth/signup`** 이 DB와 일치하는지.

## 3. GitHub Actions(CI 스모크)

각 **앱 레포 루트**에 워크플로가 있다고 가정합니다.

| 레포 | 파일(예) | 내용 요약 |
| --- | --- | --- |
| `medical-service-backend` | `.github/workflows/medical-services-ci.yml` | Gradle `test` → `bootJar` → **Docker build, push 없음** |
| `medical-service-frontend` | `.github/workflows/medical-services-ci.yml` | `npm ci` → `tsc --noEmit` → `next build` → **Docker build, push 없음** |
| `ai-medicare` | `.github/workflows/medical-services-ci.yml` (또는 별도 YAML) | AI 레포 정책에 따름 |

GHCR 푸시·`cd` 잡이 **별도 브랜치 조건**으로 붙어 있다면, 각 YAML의 `if:` 를 확인합니다. 인프라 쪽 [infra.md](infra.md) **「GitHub Actions 워크플로」** 절도 함께 보면 됩니다.

## 4. 배포 직후 end-to-end 체크(짧게)

1. SSM **Success** → ALB 타깃 **healthy** (잠시 불안정 가능).  
2. 백엔드 **`/health` 200** (재시도).  
3. 프론트에서 **로그인** → 의사·간호사 **환자 조회** 등 핵심 화면.  
4. 비밀번호 재설정·메일을 쓰는 경우: **SMTP 설정**과 메일 속 **프론트 URL**이 실제 서비스 URL과 같은지.

같은 주제를 **배포·인프라 관점**에서만 정리한 문서는 [배포시_문제_해결.md](배포시_문제_해결.md)입니다.
